# 検索テスト

[← 目次](./00-index.md)

インライン検索の動作確認用です。各キーワードは文書内で一意または少数出現するよう配置しています。

## 使い方

1. `/` で前方検索、または `?` で後方検索を開始
2. クエリを入力して `Enter`
3. マッチは黄色ハイライト、現在のマッチはマゼンタ強調
4. `n` / `N`（または `Tab` / `Shift-Tab`）でマッチ間を移動
5. `Esc` で検索をクリア（終了しない）

## 検索キーワード一覧

| キーワード | 用途 | 大文字小文字 |
|-----------|------|-------------|
| `UNIQUE_ALPHA_7X` | 単一マッチ | 区別なし |
| `unique_beta` | 複数マッチ（この行と次の段落） | 区別なし |
| `UNIQUE_GAMMA` | 後方検索用 | 区別なし |
| `コードブロック` | 日本語検索 | — |
| `println` | コードブロック内テキスト | — |

### unique_beta の二つ目

この段落にも unique_beta が含まれます。`n` で次のマッチへ移動できることを確認してください。

## コードブロック内

```rust
fn search_target() {
    println!("UNIQUE_DELTA_IN_CODE");
}
```

`UNIQUE_DELTA_IN_CODE` または `println` でコード内がヒットすることを確認します。

## テーブル内

| 列 | 検索用語 |
|----|----------|
| A | TABLE_SEARCH_TERM |
| B | 通常のセル |

`TABLE_SEARCH_TERM` でテーブルセルが検索対象になることを確認します。

## 確認項目

- [ ] `/` で前方検索プロンプトが表示される
- [ ] 大文字小文字を区別しない
- [ ] マッチがハイライトされる
- [ ] `n` / `N` でマッチ間を移動しスクロールが追従する
- [ ] コードブロック・テーブル内も検索される
- [ ] `Esc` で検索がクリアされる
