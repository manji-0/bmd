# ドキュメント遷移

[← 目次](./00-index.md)

ファイルスタックとアンカースタックの動作を段階的に確認します。

## ステップ 1 — 深いリンクチェーン

次の順でリンクを辿り、各段階で `O` の戻り先を確認してください。

1. [レベル A: 見出し](./01-headings.md)
2. 開いたら [レベル B: リスト](./03-lists.md) へ
3. さらに [レベル C: Mermaid](./10-mermaid.md) へ

| 操作 | 期待結果 |
|------|----------|
| `O`（C 上） | B（リスト）に戻る |
| `O`（B 上） | A（見出し）に戻る |
| `O`（A 上） | このページ（遷移テスト）に戻る |

## ステップ 2 — アンカー優先

[見出しドキュメントのアンカー](./01-headings.md#ナビゲーション用セクション-b) へジャンプしたあと:

| 操作 | 期待結果 |
|------|----------|
| `O` | ジャンプ前のスクロール位置（このページ内） |
| `Esc` | アンカースタックをリセットしジャンプ前位置へ |

ファイルスタックとアンカースタックが両方ある場合、アンカーが優先されます。

## ステップ 3 — Esc とドキュメントスタック

1. [検索テスト](./09-search-targets.md) を開く
2. [テーブル](./06-tables.md) を開く（2 層目）
3. `Esc` — 最初に開いたファイル（このページまたは検索テスト）へ戻る挙動を確認

## スタック上限

スタックは最大 64 層です。深いネストで上限メッセージがステータスバーに表示されることを確認する場合は、循環リンクを手動で辿ってください（このテスト群では意図的に循環は作っていません）。

## 確認項目

- [ ] 相対パスで別ファイルを開ける
- [ ] `O` で一つ前のファイル／位置に戻れる
- [ ] アンカージャンプ後 `O` でスクロール位置に戻れる
- [ ] `Esc` でスタックがリセットされる
- [ ] プリフェッチにより再訪問時の表示が速い（任意）
